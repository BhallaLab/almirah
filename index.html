napi#

napi aids in the organization, maintenance, and usage of both file-based and table-based datasets. Developed to handle neuroimaging datasets, it provides data format conversion support for specific modalities.

Out-of-the-box, napi supports:

  • BIDS specification for neuroimaging datasets.

  • BIDS-like specification for genome sequence data.

  • Interfacing with databases supported by SQLAlchemy.

  • File format conversion for neuorimaging data.

  • Lazy access of DataLad compatible datasets.

With some time on config writing, napi can:

  • Organize unformatted datasets.

  • Query and filter across the dataset.

  • Migrate records from one database to another.

Installing#

napi can be installed with pip:

$ python -m pip install napi

Alternatively, you can grab the latest source code from Github:

$ git clone https://github.com/girishmm/napi.git
$ cd napi
$ pip install .

Usage#

The User Guide is the place to go to learn how to use the libray and accomplish common dataset related tasks. If you have the time, we recommend following the Tutorial for a complete overview.

napi relies heavily on configs to guide its tasks. The Writing configs page describes the possibilities in detail.

If you want to access the ADBS dataset, please head over to From access to analysis for a walk-through. The Dataset Reference documentation provides details on the files and records available in the dataset. If you have not heard of the ADBS study, but would like to know, The ADBS Dataset page can help acquaint you.

The API Reference documentation provides API-level documentation.

Can you use napi to manage your dataset? Of course. Have a look at Translate to another dataset section for files to tweak to achieve this.

Who uses napi?#

napi was primarily developed to support and maintain the data generated by the ADBS study. But we believe it can aid with other datasets too! Even non-neuroimaging ones!

If napi was able to help with your dataset, we would love to hear about it.

Translate to another dataset#

If your dataset is a BIDS dataset, congrats you are done!

If you dataset follows a different specification, describe it as a config file by referring to Specification details. This should allow filtering and querying of the dataset.

If you would like to organize data according to a described specification, define rules according to Organization rules.

If you need assistance migrating records from one database to another, or one schema to another, Database mapping can guide you.

Help and Support#

We try to make napi robust, but there might be things we missed. If it does not meet your use case or you have trouble adapting it to your dataset, please write to us or open an issue on the repository or drop by Github Discussions.
